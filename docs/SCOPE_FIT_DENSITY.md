# Scope-Fit Density Audit & Standard

## Why this exists

The archive should not force every universe into one payload density template. Scope-fit modeling means compact systems stay compact while broad systems model enough core nodes to remain honest.

## Representative audit (current universes)

### Most compressed / under-modeled risks

- **Death Note** (`node-graph`): only 8 characters for a control-network thesis with 15 relationships and 5 factions. Core web is strong, but cast may under-represent institutional pivots.
- **Solo Leveling** (`node-graph`): 8 characters / 8 relationships in a sovereignty + guild pressure system suggests compressed network modeling.
- **Chainsaw Man** (`node-graph`): 8 characters / 4 factions; acceptable for current arc slice, but easy to under-model if added arcs are not represented.
- **Goblin Slayer** (`node-graph`): 8 characters / 4 factions can flatten guild/economic topology if key operational nodes are omitted.

### Better scope-fit baselines

- **Code Geass**: 12 characters, 6 factions, 16 relationships, and deeper anomalies/causal events shows broad-network proportionality.
- **MHA**: 10 characters, 5 factions, 17 relationships gives stronger distributed coverage.
- **Steins;Gate**: stays compact but preserves causal depth (`causalEvents` 8) aligned with timeline thesis.

### Mixed but acceptable

- **FMAB**, **HxH**, **Frieren**, **Mushoku Tensei**: generally coherent, with room for targeted enrichments only when a missing node is clearly system-critical.

## Scope-fit standard

Use one authoring class as a drafting guide (heuristic, not hard rule):

- **tight**: concentrated cast, strong rule/causal center
- **medium**: balanced cast + institutions
- **broad**: distributed institutions/network control requires wider cast/faction coverage
- **hybrid**: broad institutional spread plus deep causal/rule chains

## Practical targets (heuristics)

- tight: ~6-9 characters; keep only high-leverage nodes
- medium: ~8-11 characters; 4-6 factions
- broad: ~9-14 characters; 5-9 factions; node-heavy systems often need 12+ relationships
- hybrid: broad-network thresholds plus 5+ causal events and strong rule depth when causality is core

## Anti-bloat rule

Add an entry only when it is a:

1. control node,
2. rule enforcer/breaker,
3. causal hinge,
4. counterplay gatekeeper.

If none apply, omit it.
